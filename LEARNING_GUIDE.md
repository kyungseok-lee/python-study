# 📖 학습 가이드 (Learning Guide)

Python 학습을 효과적으로 진행하기 위한 가이드입니다.

## 🗺️ 전체 로드맵

```
┌─────────────────────────────────────────────────────────────┐
│  🌱 초급 (4주)         Python 기초 문법과 핵심 개념             │

├─────────────────────────────────────────────────────────────┤
│  🌳 고급 (8주)         심화 주제                              │
│  ├── 11주차: 데코레이터 심화                                   │
│  ├── 12주차: 메타클래스/디스크립터                             │
```

## 📅 주간 학습 계획 (예시)

### 평일 (월-금)
- 매일 30-60분
- 1개 챕터의 일부를 학습
- 예제 코드 실행
- 작은 문제 1-2개 풀이

### 주말
- 1-2시간
- 챕터 완성
- 큰 문제 풀이
- 작은 프로젝트 적용

### 매월 1회
- 복습 + 빠진 부분 보충
- 미니 프로젝트 진행

## 📝 학습 방법론

### Active Learning (능동적 학습) ✅

#### Do
- 코드를 **직접 타이핑**하기
- 결과를 **예측하고 실행**해 확인
- 변형 만들기 ("이렇게 바꾸면?")
- **소리내어 설명**해보기
- 작은 프로젝트에 **즉시 적용**

#### Don't
- 단순히 읽기만 하기
- 복사-붙여넣기만 하기
- 너무 빨리 진도 나가기
- 이해 못한 채 넘어가기

### Spaced Repetition (간격 반복)
- 새로 배운 내용을 1일 후 복습
- 1주일 후 다시 복습
- 1개월 후 또 복습

### Project-Based Learning (프로젝트 기반)
이론만으론 부족합니다. 매 단계마다 작은 프로젝트:
- 초급: 계산기, 메모장
- 중급: 가계부, 블로그 시스템
- 고급: 비동기 다운로더, 캐시 라이브러리
- 전문가: 웹 앱, CLI 도구

## 💡 막혔을 때

### 단계별 해결법
1. **다시 읽기**: 챕터를 처음부터 천천히
2. **작게 분해**: 큰 문제를 작은 단계로
3. **REPL 활용**: 한 줄씩 인터프리터에서 실험
4. **출력 추가**: `print()` 로 변수 상태 확인
5. **공식 문서**: docs.python.org
6. **검색**: "python how to ..."
7. **AI/커뮤니티**: ChatGPT, Stack Overflow
8. **잠시 쉬기**: 머리 비우고 다시 보기

## 🏆 성장의 신호

### 처음 (Novice)
- 문법을 외우려고 함
- 코드를 복사해서 씀
- 에러 메시지가 무섭다

### 초보 (Beginner)
- 간단한 코드를 직접 작성
- 검색해서 답을 찾음
- 에러 메시지를 읽기 시작

### 중급 (Intermediate)
- 패턴을 알아본다
- 라이브러리를 조합한다
- 디버깅이 자연스럽다

### 고급 (Advanced)
- 코드 품질을 신경 쓴다
- 트레이드오프를 이해한다
- 다른 사람의 코드를 리뷰한다

### 전문가 (Expert)
- 새로운 라이브러리/패턴을 만든다
- 멘토링을 한다
- 커뮤니티에 기여한다
